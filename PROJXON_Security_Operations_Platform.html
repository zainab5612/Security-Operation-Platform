The Threat Detection Engine

Identity & Access Detections
  • Repeated sign-in failures followed by success.
  • Sign-ins originating from unusual locations ("Impossible Travel").
  • Multi-factor authentication (MFA) changes or disabling actions.
  • Privilege escalations (New Super Admin assignments).
SaaS & Data Exfiltration Detections
  • Unapproved third-party OAuth applications requesting sensitive scopes.
  • Massive document downloads from Google Drive.
  • Internal folders updated to allow public external sharing links.
  • New external auto-forwarding rules created inside Gmail accounts.
Rules mature across four methods: Threshold-Based logic, Behavioral Baselines, Machine-Learning-Assisted Anomaly Detection, and Cross-System Correlation.
12

Anomaly Detection Defended

Established Baseline Patterns

The core engine analyzes daily system usage to record normal operational bounds:

  • Standard operational business hours by department.
  • Recognized corporate and personal hardware devices.
  • Expected data download thresholds per worker profile.
  • Common administrative system change patterns.
Identified Behavioral Anomalies

The platform raises an investigation lead when activity deviates from the baseline:

  • User signs into accounts at 03:00 AM local time.
  • An analyst pulls down 10x their standard daily file volume.
  • A manager updates sensitive permissions outside regular cycles.
  • Access attempts hit systems via completely unmapped browsers.
Operational Rule: Anomaly detection highlights behavioral shifts. It serves as an investigative lead, not a definitive confirmation of compromise.
Machine learning layer: After enough baseline data is collected, ML-assisted detection can help identify unusual login times, abnormal file downloads, abnormal OAuth grants, and unusual Orkavault access patterns.
